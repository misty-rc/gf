# gf — Go製 高速ファイル検索コマンド

## 概要

`gf` はファイル名を対象とした高速 CLI 検索ツールです。
goroutine による並行ディレクトリ走査を採用し、大規模ディレクトリツリーでも
高いスループットを維持します。グロブ・正規表現の両方によるパターンマッチをサポートします。
公開を見据えたシングルバイナリとして配布可能です。

## 使い方

```
gf [オプション] [パターン] [検索ディレクトリ]

# 例
gf '*.go'                     # .go ファイル（グロブ、デフォルト）
gf -r '\.go$'                 # .go ファイル（正規表現）
gf 'main*' -e go              # 拡張子 go で main から始まるファイル
gf -t d 'cache*'              # cache から始まるディレクトリのみ
gf -t f --hidden '.env*'      # 隠しファイルを含む .env* ファイル
gf --depth 2 '*.md' ./docs    # docs/ 以下 2階層まで
gf '*.go' --sort              # ソートして出力
```

## 機能要件

### 検索

- カレントディレクトリ（または指定ディレクトリ）を再帰的に検索する
- パターンはファイル名のみに適用（パス全体ではない）
- パターンを省略した場合はすべてにマッチ
- goroutine + worker pool による並行走査

### パターンマッチ

| モード | オプション | 例 |
|---|---|---|
| グロブ（デフォルト） | なし | `*.go`, `main*`, `?oo` |
| 正規表現 | `-r`, `--regex` | `\.go$`, `^main` |

- グロブは Go 標準の `path.Match` 準拠
- 正規表現は Go の `regexp` パッケージ準拠
- 両方同時指定はエラー

### フィルタリング

| オプション | 説明 |
|---|---|
| `-t`, `--type` | `f`=ファイル, `d`=ディレクトリ |
| `-e`, `--ext` | 拡張子でフィルタ（ドットなし。例: `go`, `md`） |
| `--hidden` | 隠しファイル・ディレクトリを含める（デフォルト: 除外） |
| `--depth N` | 最大探索深さ（デフォルト: 無制限） |

### 出力

| オプション | 説明 |
|---|---|
| `--sort` | パスをアルファベット順にソートして出力 |

- マッチしたパスを1行ずつ標準出力に出力
- パスはスラッシュ区切り（クロスプラットフォーム統一）
- デフォルトは順序不定（並行処理のため）
- `--sort` 指定時は全件収集後にソート（メモリ使用量増加のトレードオフあり）
- 見つからない場合は何も出力せず終了コード 0

### エラー処理

- 検索ディレクトリが存在しない: stderr にエラー、終了コード 1
- 不正な正規表現: stderr にエラー、終了コード 2
- グロブと正規表現の同時指定: stderr にエラー、終了コード 2
- アクセス権限エラー: スキップして続行
- 不正なオプション: 使い方を表示して終了コード 2

## 技術スタック

| 項目 | 内容 |
|---|---|
| 言語 | Go 1.22+ |
| CLIライブラリ | 標準 `flag` パッケージ（起動コスト最小化） |
| 主要標準pkg | `path/filepath`, `io/fs`, `path`, `regexp`, `sync`, `sort` |

## パフォーマンス設計

- `filepath.WalkDir` をベースに使用（`Walk` より効率的）
- ディレクトリを goroutine で並行処理（worker pool パターン）
- 正規表現は起動時に1回コンパイルして使い回す
- 結果は channel 経由で収集し、メインゴルーチンが出力

## ビルド・実行要件

- `go build` 一発でシングルバイナリ生成
- Windows / macOS / Linux 対応
- Go 1.22 以上を前提

## ファイル構成

```
gf/
├── main.go            # エントリーポイント・フラグ定義
├── search/
│   └── search.go      # 並行検索ロジック
├── go.mod
├── SPEC.md
└── .gitignore
```
