# フラグ仕様

## フラグ一覧

| フラグ | 説明 |
|---|---|
| `-r` / `--regex` | 正規表現モード |
| `-p` / `--path` | フルパスにパターン適用 |
| `-t f\|d` / `--type` | ファイル/ディレクトリフィルタ |
| `-e EXT` / `--ext` | 拡張子フィルタ |
| `--hidden` | 隠しファイル含む |
| `--depth N` | 最大探索深さ |
| `--sort` | ソート出力 |
| `-n N` / `--limit` | 最大件数 |
| `--exclude PAT` | 除外パターン（複数可） |
| `--newer DATE` | 更新日時 以降 |
| `--older DATE` | 更新日時 以前 |
| `--created-after DATE` | 作成日時 以降（Windows/macOS のみ） |
| `--created-before DATE` | 作成日時 以前（Windows/macOS のみ） |

日付フォーマット: `YYYY-MM-DD` または `YYYY-MM-DDTHH:MM:SS`

## フラグパーサの工夫

標準 `flag` パッケージは最初の非フラグ引数でパース停止するため、
起動時にフラグ引数と位置引数を前処理で分離してから `Parse()` する。

## パターン仕様

- グロブ記号なし（`*?[` を含まない）パターン → 自動で `*pattern*`（部分一致）に変換
- `--exclude` はグロブのみ（正規表現不可）
- ディレクトリが `--exclude` にマッチした場合はサブツリーごとスキップ

## 出力仕様

- 1行1パス、スラッシュ区切り（Windows でも `/`）
- 順序不定（並行処理のため）、`--sort` でアルファベット順
- エラー終了コード: 1 = ディレクトリ不存在、2 = 不正パターン/オプション
